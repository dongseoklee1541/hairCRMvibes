# R-08 Service Master

## 상태
- Planned
- 브랜치: `feature/r08-service-master`
- 최종 업데이트: 2026-07-07

## 목표
- 자유입력 시술명을 서비스 마스터 데이터로 표준화합니다.
- 서비스별 가격, 기본 소요시간, 활성 여부, 정렬 순서를 관리합니다.
- 예약 생성/수정과 통계가 동일한 서비스 기준을 사용하도록 합니다.

## 선행조건
- R-05 설정 페이지의 기본 시술/소요시간 구현이 main에 반영되어 있어야 합니다.
- 기존 `salon_service_defaults`를 확장할지, 새 `services` 테이블로 분리할지 결정 필요.
- 가격 필드의 통화/부가세/할인 처리 범위 확인 필요.

## 완료 기준
- 서비스 마스터 데이터 모델과 RLS 정책 구현
- 설정 또는 별도 관리 화면에서 owner가 서비스 CRUD 가능
- staff는 예약 생성/수정에서 활성 서비스 조회 가능
- 예약에 서비스 식별자와 표시명/가격 snapshot 정책이 적용됨
- `schema.sql`과 migration 동기화

## 검증 계획
- `npm run build`
- owner/staff 서비스 조회/관리 권한 smoke
- 예약 생성/수정에서 활성/비활성 서비스 선택 회귀 검증
- 모바일 390x844, 360x800 서비스 관리/예약 화면 확인

## 남은 리스크
- 기존 텍스트 `appointments.service`와 새 서비스 ID의 마이그레이션 전략이 필요합니다.
- 가격 snapshot을 저장하지 않으면 과거 매출 통계가 현재 가격 변경에 흔들릴 수 있습니다.
