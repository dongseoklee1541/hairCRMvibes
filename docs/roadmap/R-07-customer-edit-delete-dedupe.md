# R-07 Customer Edit Delete Dedupe

## 상태
- Planned
- 브랜치: `feature/r07-customer-edit-delete-dedupe`
- 최종 업데이트: 2026-07-07

## 목표
- 고객 상세에서 이름/전화번호/메모를 수정할 수 있게 합니다.
- 고객 삭제 또는 비활성화 정책을 정하고 UI에 반영합니다.
- 전화번호/이름 기반 중복 고객 탐지와 정리 플로우를 마련합니다.

## 선행조건
- R-01 RLS 정책이 main에 반영되어 있어야 합니다.
- 개인정보 보존/삭제 정책 결정 필요: hard delete, soft delete, merge 중 선택.
- 중복 판단 기준 확인 필요: 전화번호 정규화, 이름 유사도, 수기 병합 범위.

## 완료 기준
- 고객 상세/목록에서 편집 진입 가능
- 고객 수정 폼의 loading/error/empty 상태 제공
- 삭제 또는 비활성화 시 예약/시술 이력 보존 규칙이 명확함
- 중복 후보를 보여주고 사용자가 안전하게 정리할 수 있음
- owner/staff 권한별 허용 범위가 RLS와 UI에서 일관됨

## 검증 계획
- `npm run build`
- owner/staff 고객 조회/수정/삭제 또는 비활성화 smoke
- 예약 이력이 있는 고객의 삭제/비활성화 회귀 검증
- 모바일 390x844, 360x800 고객 상세/편집 viewport 확인

## 남은 리스크
- 고객 삭제는 예약 이력 보존과 충돌할 수 있으므로 soft delete가 기본 후보입니다.
- 중복 병합은 데이터 손실 위험이 있어 preview/confirm 단계를 분리해야 합니다.
