# R-09 Stats Advanced

## 상태
- Planned
- 브랜치: `feature/r09-stats-advanced`
- 최종 업데이트: 2026-07-07

## 목표
- 기존 예약 건수 중심 통계를 매출, 객단가, 재방문율 지표로 확장합니다.
- R-04 KST date key와 R-08 서비스/가격 데이터를 기준으로 통계 계산을 일관화합니다.
- 모바일에서 빠르게 훑을 수 있는 운영 지표 화면을 제공합니다.

## 선행조건
- R-04 KST 날짜 유틸이 main에 반영되어 있어야 합니다.
- R-08 서비스 마스터와 가격 snapshot 정책이 먼저 결정되어야 합니다.
- 매출 기준 확인 필요: 완료 예약만 집계할지, 취소/노쇼/할인 처리 범위.

## 완료 기준
- 월간 매출, 객단가, 시술별 매출/건수, 재방문율 지표 제공
- KST 기준 월/일 범위로 집계
- 빈 데이터/로딩/오류 상태 제공
- owner/staff 통계 조회 권한이 RLS와 UI에서 일관됨
- `npm run build` 통과

## 검증 계획
- `npm run build`
- 월 경계/KST 날짜 회귀 검증
- 완료/취소/확정 상태별 집계 제외/포함 검증
- 모바일 390x844, 360x800 통계 화면 viewport 확인

## 남은 리스크
- 가격 snapshot 없이 통계를 만들면 과거 데이터가 재계산될 수 있습니다.
- 재방문율 정의(기간, 고객 기준, 첫 방문 제외 여부)는 구현 전 확정 필요입니다.
